This project aims to design and implement a mobile game application to assist 
beginner and intermediate Chinese learners in reinforcing their language skills 
in an engaging and interactive way. The app will include 3 mini-games, each 
targeting a specific aspect of Chinese language learning, such as vocabulary, 
pronunciation, character recognition, and grammar. A scoring and reward 
system will be integrated to motivate users and track their progress. The 
application will be developed for the Android platform using Unity and Android 
SDK. 
 
1. An initial analysis of user requirements, and how data will be 
collected.  
a. Game Initialization  
The user is able to login/register to the app and see a main menu offering access 
to different games, user profile, and settings.  
b. User Progress Tracking 
Users shall be able to view their profiles, game scores, historical performance, 
and unlocked achievements. 
c. Game Selection Interface 
1） Users will be able to choose from three Chinese language games, each with 
clear instructions and difficulty settings. 
2） Users shall be able to select and adjust difficulty levels, details can be seen 
in item f of the requirements. 
d. Game Type 
1) Character Matching Game: Users match Pinyin to the correct Chinese 
character. The game will use a drag-and-drop or tap-to-match interface. 
2) Pronunciation Quiz: Users need to pronounce the words and then get the 
correct pronunciation and receive instant feedback. 
3) Word Puzzle: Users will rearrange scrambled characters to form correct 
words or sentences. 
e. Reward System 
Users can earn a variety of virtual badges by completing challenges across the 
application.  
1) They can unlock Milestone Badges by reaching cumulative targets, such as 
"First Steps" after completing 10 games or "Login Streak" for using the app 
for 7 consecutive days.  
2) They can unlock Game-Specific Badges by completing the game in 
particular conditions, like "Perfect Match" for achieving a 10-streak in

f. Difficulty System 
Users can select from three standardized difficulty levels (Easy, Medium, Hard) 
across all three game types.  
 
2. The algorithms, methodologies, and other techniques to be 
employed. 
a. Character Matching Game 
Randomized Item Selection Algorithm: Get random characters and distractors 
from the database. 
b. Pronunciation Quiz 
Speech Recognition and Pronunciation Assessment: Integrate third-party speech 
evaluation SDKs to analyse users' pronunciation through multi-dimensional 
metrics including phoneme accuracy, tone correctness, fluency, and 
completeness.  
c. Word Puzzle 
NLP Processing: Use NLP models to segment sentences based on parts of 
speech and number them. 
Shuffling Algorithm: Randomize the numbers created by the NLP model to 
ensure a different puzzle sequence is generated each time. 
 
3. An initial specification of how users will interact with the system.  
User Action:  
1) The user taps on the icons for the four games (Character Matching, 
Pronunciation Quiz, Word Puzzle, AI Painting Master), profile, or settings 
on the main screen. 
2) On a game's pre-game screen, the user swipes left or right to view 
instructions and taps to select a difficulty level (Easy, Medium, Hard). 
System Feedback:  
1) The tapped icon is highlighted visually, and the interface immediately 
transitions to the corresponding game menu or functional screen. 
2) The screen content updates with the swipe, and the selected difficulty button 
changes color or state to confirm the choice. 
 
4. Programming language / database/ software package and 
hardware to be used 
The application will be developed using the Unity Engine as the primary 
development platform, with all core game logic and interactions programmed in 
C#. Targeting the Android platform, the project will utilize SQLite for data 
storage to manage user progress, game states, and scores. Furthermore, a third-
party commercial speech evaluation SDK (e.g., iFLYTEK, Tencent Cloud) will 
be integrated to analyse users' pronunciation, and NLP model to segment 
sentences.

Summary of ethical issues: 
In accordance with the project ethics guidance, the following ethical 
considerations have been identified and addressed for the "Android Chinese 
Words Game App" project: 
1. Human Participants: 
This project does not involve human participants in surveys, interviews, or 
structured testing during development, so informed consent and debriefing 
procedures are not applicable.  
2. Artificial Intelligence (AI) Techniques: 
This project utilizes third-party AI services, including a speech evaluation SDK 
and an NLP model. The following ethical considerations have been addressed: 
a. Data Privacy 
Speech Data: Users will be clearly notified that voice inputs are processed 
by a third-party service for real-time evaluation. No speech data is stored 
locally or by the developer. 
Implementation: A privacy notice will be displayed within the app, 
explaining data processing practices and providing links to third-party 
privacy policies. 
b. Algorithmic Bias 
Speech Recognition: The accuracy of pronunciation assessment depends 
on the speech SDK's training data and language support. Potential biases in 
recognizing non-native accents or regional variations are acknowledged. 
NLP Processing: The NLP model may have biases in handling regional 
expressions or colloquialisms 
c. Documentation and Transparency 
All AI components will be clearly documented in the final report, 
including: 
1) API specifications and service providers 
2) Limitations in recognition accuracy and content generation 
These measures ensure responsible development and deployment of AI 
technologies while maintaining focus on educational benefits and user 
protection.
